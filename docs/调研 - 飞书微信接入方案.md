# OpenClaw 接入飞书/微信调研方案

## 1. 调研结论

### 1.1 支持情况一览

| 渠道 | OpenClaw 原生支持 | 配置难度 | 推荐度 |
|------|------------------|----------|--------|
| **飞书 (Feishu)** | ✅ 支持 (需安装插件) | ⭐⭐⭐ 中等 | ⭐⭐⭐⭐ 推荐 |
| **企业微信 (WeCom)** | ❌ 不支持 | - | - |
| **个人微信 (WeChat)** | ❌ 不支持 | - | - |
| Telegram | ✅ 原生支持 | ⭐⭐ 简单 | ⭐⭐⭐⭐⭐ 最推荐 |
| Discord | ✅ 原生支持 | ⭐⭐ 简单 | ⭐⭐⭐⭐ 推荐 |
| Slack | ✅ 原生支持 | ⭐⭐⭐ 中等 | ⭐⭐⭐⭐ 推荐 |
| WhatsApp | ✅ 原生支持 | ⭐⭐⭐⭐ 较复杂 | ⭐⭐⭐ |

### 1.2 微信接入现状

**重要发现**：OpenClaw **不支持个人微信和企业微信** 原生接入。

原因分析：
1. 微信个人号没有官方开放 API
2. 企业微信需要企业认证且 API 复杂
3. 第三方方案 (如 Wechaty) 稳定性差，易被封号

**替代方案**：
- 国内用户 → **飞书 (Feishu)** 或 **Telegram**
- 企业用户 → **飞书** / **Slack** / **Microsoft Teams**
- 个人用户 → **Telegram** / **Discord** / **WebChat**

## 2. 飞书接入方案

### 2.1 配置流程

#### 步骤 1: 安装飞书插件

```bash
openclaw plugins install @openclaw/feishu
```

#### 步骤 2: 创建飞书机器人

1. 访问 [飞书开放平台](https://open.feishu.cn/app) 并登录
2. 点击 **创建企业应用**
3. 填写应用名称、描述、图标
4. 从 **凭证与基础信息** 页面复制：
   - **App ID** (格式：`cli_xxx`)
   - **App Secret** (保密！)

#### 步骤 3: 配置权限

在权限管理页面批量导入以下权限：
```
message:read, message:write, im:chat, im:message,
im:resource, contact:tenant:user, contact:tenant:department
```

#### 步骤 4: 配置事件订阅

1. 开通事件订阅功能
2. 选择订阅模式：
   - **推荐**: WebSocket (无需公网)
   - 可选：Webhook (需要公网 URL)
3. 订阅事件：
   - `im.message.receive_v1`
   - `im.message.group_at_msg_v1`

#### 步骤 5: 发布应用

1. 填写应用可见范围
2. 提交审核 (通常需要 1-2 个工作日)
3. 审核通过后即可使用

### 2.2 OpenClaw 配置

运行配置命令：
```bash
openclaw channels add
```

选择 **Feishu**，然后输入：
- App ID
- App Secret

配置文件位置：`~/.openclaw/config/channels.yml`

```yaml
channels:
  feishu:
    enabled: true
    domain: "feishu"  # 或 "lark" (国际版)
    connectionMode: "websocket"
    defaultAccount: "default"
    dmPolicy: "pairing"
    accounts:
      default:
        appId: "cli_xxxxxxxxxxxxx"
        appSecret: "${FEISHU_APP_SECRET}"
```

### 2.3 配置后验证

```bash
# 检查网关状态
openclaw gateway status

# 查看日志
openclaw logs --follow

# 发送测试消息
openclaw message send --channel feishu --to "ou_xxxxx" --message "Hello from OpenClaw"
```

### 2.4 所需凭证汇总

| 凭证名称 | 获取方式 | 保密级别 |
|----------|----------|----------|
| App ID | 飞书开放平台 - 凭证与基础信息 | 公开 |
| App Secret | 飞书开放平台 - 凭证与基础信息 | 🔒 机密 |
| OpenID (用户 ID) | 飞书 API 查询或消息回调 | 公开 |
| ChatID (群组 ID) | 飞书 API 查询或消息回调 | 公开 |

## 3. 微信替代方案

由于 OpenClaw 不支持微信，以下是可行的替代方案：

### 3.1 方案对比

| 方案 | 优点 | 缺点 | 推荐度 |
|------|------|------|--------|
| **飞书** | 官方 API 支持，稳定，免费 | 需企业认证，有审核流程 | ⭐⭐⭐⭐ |
| **Telegram** | 配置简单，功能完整，免费 | 需要科学上网 | ⭐⭐⭐⭐⭐ |
| **Discord** | 配置简单，社区活跃 | 需要科学上网 | ⭐⭐⭐⭐ |
| **企业微信** | 国内可用，企业支持 | 需企业认证，OpenClaw 不支持 | ⭐⭐ |
| **WebChat** | 内置网页聊天界面 | 需自己访问网页 | ⭐⭐⭐ |

### 3.2 Telegram 配置 (最简方案)

**优势**：
- 只需一个 Bot Token
- 无需审核，即时可用
- 功能完整 (文本/图片/文件/语音)

**配置步骤**：

1. 在 Telegram 搜索 `@BotFather`
2. 发送 `/newbot` 创建机器人
3. 获取 Token (格式：`123456789:ABCxxxxxxxxxxxxxxxxxxxx`)
4. 在 OpenClaw 中配置：

```bash
openclaw channels add
```

选择 **Telegram**，输入 Token 即可。

```yaml
channels:
  telegram:
    enabled: true
    defaultAccount: "default"
    accounts:
      default:
        token: "${TELEGRAM_BOT_TOKEN}"
```

## 8. CloudClaw 插件管理 CLI 设计

### 8.1 命令设计

```bash
# 查看已安装的插件
cloudclaw plugins list

# 安装插件
cloudclaw plugins install feishu
cloudclaw plugins install telegram
cloudclaw plugins install discord

# 卸载插件
cloudclaw plugins uninstall feishu

# 查看插件状态
cloudclaw plugins status
```

### 8.2 实现方案

在 `cmd/cloudclaw/main.go` 中添加 plugins 子命令：

```go
import "cloudclaw/internal/deploy"

func main() {
    app := cli.NewApp()
    app.Commands = []cli.Command{
        // ... 现有命令
        {
            Name:  "plugins",
            Usage: "管理 OpenClaw 插件",
            Subcommands: []cli.Command{
                {
                    Name:  "list",
                    Usage: "列出已安装的插件",
                    Action: func(c *cli.Context) error {
                        return deploy.ListPlugins()
                    },
                },
                {
                    Name:  "install",
                    Usage: "安装插件",
                    Action: func(c *cli.Context) error {
                        pluginName := c.Args().Get(0)
                        return deploy.InstallPlugin(pluginName)
                    },
                },
            },
        },
    }
}
```

## 10. 插件安装后需要重启 Gateway 吗？

**是的，需要重启。**

### 原因

1. **插件是进程内加载 (in-process)**：插件代码在 Gateway 进程中运行，不是独立服务
2. **代码需要重新加载**：新插件的代码需要在启动时被加载到内存
3. **配置验证在启动时进行**：Unknown plugin ids 会被视为错误

### 完整流程

```bash
# 1. 安装插件
openclaw plugins install @openclaw/feishu

# 2. 配置插件
# 编辑 ~/.openclaw/config/plugins.yml 或 channels.yml
# 添加 App ID 和 App Secret

# 3. 重启 Gateway（必须！）
openclaw gateway restart
```

### CloudClaw 自动处理

在 CloudClaw 的 `plugins install` 命令中可以自动处理重启：

```
$ cloudclaw plugins install feishu

正在安装 @openclaw/feishu...
✓ 插件安装完成

📱 安装飞书插件后需要重启 Gateway 服务

是否立即重启？[Y/n]: Y

正在重启 Gateway...
✓ Gateway 已重启
```
